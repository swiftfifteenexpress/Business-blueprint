The automated chain that fires in milliseconds

1

Form data is encoded and sent to Make.com

Our JavaScript collects all 10 fields and sends them to the Make.com webhook URL as application/x-www-form-urlencoded — this format sends each field as a separate key-value pair so Make.com can read and map every individual field without any parsing.

POST RequestMake.com Webhook
2

Meta Pixel fires Lead event

Simultaneously, a fbq('track', 'Lead') call fires. Facebook records this person as a verified lead. This trains the ad algorithm to find more people like them and is used to optimise our Lead Generation campaigns.

Meta PixelLead Event

Make.com receives the data and routes it

Make.com's webhook trigger activates 3 parallel scenarios: (1) Add a new row to the Leads tab in Google Sheets with all 10 fields mapped to the correct columns. (2) Send an alert email to swiftfifteenexpress@gmail.com. (3) Send an automated SMS and email to the lead.

Make.comGoogle SheetsEmailSMS

Success screen shows order form button

The lead form hides and a success message appears with a direct link to swiftfifteenexpress.com/order. The WhatsApp button is a secondary option for leads who have questions before ordering. The primary path pushes them straight to place an order.

Stage 5 — Address to Coordinates (Geocoding)

Why we need to convert addresses and exactly how it works

Why geocoding is necessary: The Kwikpik API does not accept plain text addresses like "12 Allen Avenue, Ikeja". It requires precise GPS coordinates — latitude and longitude numbers. We use Google Maps to translate what the customer types into coordinates Kwikpik can understand.

Geocoding Step by Step

What our Vercel estimate.js function does when addresses are entered

1

Customer finishes typing delivery address and clicks away

The onblur event on the delivery address field triggers the fetchEstimate() function automatically. No button press needed.

2

Browser sends both addresses to our Vercel function

A POST request goes to https://swiftfifteen-api.vercel.app/api/estimate with the pickup and delivery address text. The browser never contacts Google Maps or Kwikpik directly — everything goes through our Vercel function which keeps API keys hidden.

Vercel FunctionAPI Key Protected
3

Vercel calls Google Maps Geocoding API for both addresses simultaneously

Two parallel calls go to maps.googleapis.com/maps/api/geocode/json with the Nigeria country filter applied. Google returns precise latitude and longitude for each address. Example result for "12 Allen Avenue, Ikeja":

JSON — Google Maps Response
{
  "status": "OK",
  "results": [{
    "geometry": {
      "location": {
        "lat": 6.5974061,
        "lng": 3.3548596
      }
    }
  }]
}
4

Coordinates are sent to Kwikpik for price calculation

Vercel now builds the Kwikpik payload with the correct nested structure the API requires and sends it to api.kwikpik.io/partners/requests/estimate

JSON — Sent to Kwikpik
{
  "vehicleType": "motorcycle",
  "insured": false,
  "pickupLocation": {
    "latitude":  6.5974061,
    "longitude": 3.3548596,
    "address":   "12 Allen Avenue, Ikeja, Lagos"
  },
  "deliveryLocation": {
    "latitude":  6.448041,
    "longitude": 3.480237,
    "address":   "10 Admiralty Way, Lekki Phase 1"
  }
}
5

Kwikpik returns raw delivery cost

Kwikpik calculates the real distance cost based on their rider network and traffic data. Our function applies 25% margin on top of whatever Kwikpik charges. If launch offer is active (June 1–3), an additional 20% discount is applied. The final customer price is sent back to the browser.

JSON — Our response to the browser
{
  "success":       true,
  "kwikpik_price": 13975,   // what we pay Kwikpik
  "your_price":    17500,   // what customer pays (+25%)
  "duration":      "31 mins",
  "fallback":      false
}

Stage 6 — Payment Collection

How Paystack collects money and how we verify it before doing anything

Why We Verify Before Dispatching

The critical security check that prevents fake orders

The problem we solved: In our original design, clicking "Confirm Order" would dispatch a real Kwikpik rider even if no payment was made. Anyone could get a free delivery. Our current architecture makes it impossible to dispatch without a verified Paystack payment. The Kwikpik API is never called unless Paystack confirms the transaction as "success".

1

Customer clicks "Pay Now"

The Paystack Popup opens directly on our page — customer never leaves swiftfifteenexpress.com. They can pay by card, bank transfer, or USSD. Our public key pk_live_c8e1... initialises the popup. The amount in kobo (NGN × 100) and their email are passed automatically.

2

Customer completes payment on Paystack

Paystack handles all payment processing securely. When payment succeeds, Paystack calls our callback function with a unique payment reference — a string like SFX-1748123456789. If the customer closes the popup without paying, nothing happens.

3

We verify the reference with Paystack's API

Our verify-payment.js Vercel function calls api.paystack.co/transaction/verify/{reference} using our secret key. This is the key security step — we ask Paystack directly "did this payment actually succeed?" before we do anything.

JSON — Paystack verification response
{
  "status": true,
  "data": {
    "status":   "success",   // must be "success"
    "amount":   1750000,    // ₦17,500 in kobo
    "reference": "SFX-1748123456789",
    "customer": {
      "email": "customer@email.com"
    }
  }
}
4

Only after "success" — Kwikpik dispatch fires

If Paystack returns "status": "success", the same Vercel function immediately calls Kwikpik to create the delivery. If Paystack returns anything other than success, dispatch is blocked and an alert is sent to our team.

Stage 7 — Kwikpik Rider Dispatch

How the order is placed on Kwikpik and what comes back

What We Send to Kwikpik

The exact payload sent to /partners/requests/initiate

JSON — Kwikpik dispatch payload
{
  "vehicleType":           "motorcycle",
  "pickupLocation": {
    "latitude":  6.5974061,         // from Google Maps geocoding
    "longitude": 3.3548596,
    "address":   "12 Allen Avenue, Ikeja, Lagos"
  },
  "deliveryLocation": {
    "latitude":  6.448041,
    "longitude": 3.480237,
    "address":   "10 Admiralty Way, Lekki Phase 1"
  },
  "senderName":            "Emeka Okafor",
  "senderEmail":           "emeka@business.com",
  "senderPhoneNumber":     "08012345678",
  "recipientName":         "Chidi Nwosu",
  "recipientPhoneNumber":  "08098765432",
  "description":           "2 pairs of shoes in a cardboard box",
  "itemCategory":          "general",
  "itemValue":             50000,
  "itemWeight":            3,             // kg — converted from weight range
  "insured":               false,
  "itemQuantity":          1
}

Kwikpik responds with a unique request ID (result.id) which we use to build the tracking URL: kwikpik.io/track/{id}. This tracking URL is sent back to the browser, shown on the success screen, and logged to Google Sheets so Make.com can SMS it to the recipient.
